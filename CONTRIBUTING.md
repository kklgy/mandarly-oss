# Contributing

Thanks for helping improve Mandarly OSS.

## Scope

Good contributions include:

- fixes for booking, payment, settlement, referral, and classroom workflows
- tests for domain services and integration boundaries
- frontend accessibility, i18n, and responsive layout improvements
- local setup and documentation improvements
- security hardening that does not require production secrets

Out of scope:

- production credentials or cloud account details
- customer-specific documents
- private deployment scripts
- real user data

## Development Rules

1. Keep examples local-first.
2. Use placeholders for provider credentials.
3. Add focused tests when changing backend domain behavior.
4. Keep pull requests small enough to review.
5. Do not commit generated build output.
