# Mandarly 阿语 i18n 术语词典(MSA)

> 创建日期:2026-05-19
> 范围:`app/src/i18n/locales/ar.js` 及衍生文件
> 标准:现代标准阿拉伯语(فصحى عصرية / Modern Standard Arabic),对标 Cambly / Preply / iTalki 阿语版
> 维护规则:新增 i18n key 含本表术语时,直接套用,避免漂移。本表外的新词,先讨论再增补。

## 一、设计原则

1. **MSA 优先**:统一书面语调,覆盖海湾 + 北非 + 黎凡特用户;不掺方言(Khaleeji / Egyptian / Levantine)。
2. **品牌词不译**:`Mandarly` / `Google` / `Apple` / `WhatsApp` / `WeChat` / `Stripe` / `LCIC` / `HSK` 等保留拉丁字符或商标原文。
3. **货币 / 数字**:UI 数字保留西方阿拉伯数字(0-9),不切换东方数字(٠-٩);货币代码(HKD/USD/CNY)保留拉丁字符。
4. **变量插值保留**:`{name}` / `{n}` / `{date}` / `{tz}` 等插值占位符**原样保留**,不翻译,不加空格。
5. **占位符语境**:`{n} 节课`、`{count} 条评价` 等阿语单复数视语境用 MSA 通用单数形式,避免双复数复杂分支(教育产品在 UAE 通用做法)。
6. **标点**:阿语句末用阿语逗号 `،` 和阿语问号 `؟`,但 URL / Email 中保留拉丁标点。

## 三、阿语写作规范

### 标点

- 句末逗号:`،`(不是 `,`)
- 句末问号:`؟`(不是 `?`)
- 句末分号:`؛`(不是 `;`)
- URL / Email / 时间 24h 格式:保留拉丁标点

### 变量插值规则

- `{name}` `{n}` `{date}` `{tz}` `{count}` `{amount}` 等**原样保留**,前后**不加空格**(阿语字符自动正确显示)
- 含变量的复合句:`الموعد {time}` 或 `{n} حصة`,顺序按阿语语序排列

### 容易踩坑

- ❌ 不要用 `أنت` / `أنتِ`(你 - 阳/阴)直接称呼用户,默认用动词第二人称命令式(`احجز` / `سجّل` / `اختر`),性别中立
- ❌ 不要用 `Click here`(انقر هنا)风格,UI 上下文清晰
- ✅ 错误提示用客观 + 礼貌:`"حدث خطأ. حاول مرة أخرى."`(出错了,请重试),不用 `"你失败了"`
- ✅ 引导文案用鼓励语调:`"ابدأ رحلتك"`(开始你的旅程),`"خطوة واحدة بعد"`(只差一步)

### 数字

- UI 数字默认西方阿拉伯数字(0-9):`5 درس` / `24 ساعة` / `2026-05-19`
- 仅在正式宗教/古典语境下用东方数字(٠-٩),教育互联网产品不用

### 货币

- HKD / USD / CNY 保留拉丁字符,前置或后置由 UI 决定:`HKD 60` 或 `60 HKD`
- 价格变量:`{price} {currency}` 标准模式

## 四、品牌词

| 类型 | 写法 | 备注 |
|---|---|---|
| 品牌名 | Mandarly | 保留拉丁字符,不音译为 `ماندرلي` 等(已在 footer / legal 已固化) |
| 公司主体 | ماندارلي تكنولوجي ليمتد | 法律页面音译;UI 不展示 |
| Slogan | تعلم اللغة الصينية مع معلمين أصليين 1 مقابل 1 | 已固化 |
| 教师服务 | معلم Mandarly | 保留 Mandarly 拉丁字符 |

## 五、维护

- 新增 i18n key 优先复用本表术语
- 本表外新词:在 PR 描述中标"新增 ar 术语",链接到本表,后续合并入表
- 季度回顾:如发现真实阿语用户反馈某术语生硬,在 changelog 记录后微调
- 母语校对:未来如有阿语母语者协作,基于本表做润色 PR,而非重做翻译
